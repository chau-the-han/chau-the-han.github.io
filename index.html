Long-Term Memory

Structured insights extracted from conversations and persisted across sessions.

Strategy — what to remember

  • Semantic: facts and knowledge from conversations.
  • User preference: likes, dislikes, settings ("prefers window seat").
  • Summary: per-session conversation summaries.
  • Episodic: reasoning over past episodes (with reflection).
  • Custom / override: bring your own extraction & consolidation prompts and model.

Extraction

  • Asynchronous, LLM-driven; triggered automatically after events land in short-term memory.
  • Produces structured memory records in the strategy's schema.

Consolidation

  • Retrieves semantically similar existing records, then the LLM decides:
    • ADD — information is genuinely new
    • UPDATE — complements or supersedes an existing record (recent info wins)
    • SKIP / NO-OP — redundant ("loves pizza" ≈ "likes pizza")
  • Resolves contradictions and minimizes duplicates so memory stays clean.
